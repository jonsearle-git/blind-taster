# Blind Taster — Product Specification

## Overview

A food and drink blind testing app for iOS and Android. Clean, polished, tasteful design — no tacky icons. Sold to consumers.

Players join a hosted game and answer a questionnaire once per round. The host controls the pace. Works over the internet via Cloudflare PartyKit — no sign-in required.

- When the host starts a game, the full questionnaire is uploaded to the PartyKit server, which stores it and broadcasts questions to players. The server is the source of truth for all game state.
- Questions are shown to players **one at a time** (not all at once).

---

## Modes

| Mode | Description |
|------|-------------|
| **Quiz mode** | Host provides correct answers. Players are scored. Leaderboard shown to host at end. Each player sees their own score + right/wrong breakdown. Round labels revealed to players at end. |
| **Feedback mode** | No correct answers. Host collects response data (e.g. for product research). Only the host sees results at end. Players see a custom thank you screen. |

---

## Questionnaire

- Questionnaires are created by the host and **stored locally on their device** (SQLite/AsyncStorage)
- Can be created ahead of time and reused across multiple games
- Applied once per round for every round in the game

### Question Types

| Type | Description |
|------|-------------|
| **Multiple choice — text** | Player picks one option from a list of text answers. 1 point if correct. |
| **Multiple choice — number** | Player picks one option from a list of numbers. 1 point if exact match. |
| **Slider / number input** | Same type, UI choice only. Player picks or enters a number. 1 point if exact match. |
| **Tags** | Player selects from a set of text tags (e.g. tasting notes: "grassy", "peach"). Host can limit how many tags a player can select. 1 point per correct tag selected. |
| **Price** | Player enters a monetary value (e.g. £9.99). Currency symbol set by host. 1 point if exact match. |
| **Free text** | Player types a comment / free response. **Not available in quiz mode** — feedback mode only, unscored. |

---

## Rounds

- Host selects the number of rounds (e.g. 5)
- Each round = one item being tested (e.g. Wine A, Wine B)
- Rounds are numbered (1, 2, 3…)
- Host can optionally **label each round** (e.g. "Château Margaux 2015") — this label is **hidden from players** during the game. In quiz mode it is revealed to players at the end. Host is clearly warned when creating rounds that labels are hidden until reveal.
- **Reminder in the questionnaire builder:** when the host finishes building a questionnaire, show a prompt: "Remember to label the items you are testing in the Rounds screen. In quiz mode, these labels are hidden from players until the end."
- Host manually advances players to the next round once everyone has answered
- Host can see per-player response status during a round
- Host can kick any player at any time (player is not told why, just removed)
- Kicked players can rejoin — their name is reserved

---

## Joining

- Players join via a **deep link** (`blindtaster://join/ROOMCODE`) or by entering the room code manually
- Room code is generated by PartyKit
- Host admits or denies each join request
- Host can start the game whenever they are ready

---

## Host as Player

- Host can optionally play as a player
- If the host is also a player, they count toward the "everyone has answered" check
- Host controls are accessible via a **dropdown in the top-right of the banner** — visible on all screens including the player questionnaire screen
- If the host sets correct answers, they **cannot** play as a player

---

## Disconnection & Reconnection

- If a player disconnects, the game pauses and waits
- Player can rejoin using the same room code and their reserved name
- If the host disconnects, the game pauses until the host reconnects
- Host can kick unresponsive players to unblock the game

---

## Results

### Quiz mode
- Each player sees their own score + which questions they got right/wrong + round labels revealed
- If reveal timing is set to **after each round**, the player's running score is shown in the banner on the top of the screen throughout the game
- Host sees everyone's scores and knows the winner. Each player is told their finishing position (e.g. 1st, 2nd, 3rd, 4th etc.)

### Feedback mode
- Only the host sees the collected responses
- Players see a thank you screen with a custom message set by the host when setting up the game

---

## Reveal Timing

- Host chooses: **reveal after each round** OR **reveal at the end of the game**

---

## Host Dropdown Controls

Accessible via a dropdown in the top-right of the banner on all host screens during a game.

- **Players** — list of all players with a green dot if connected, grey if disconnected. Each player has a kick icon — tapping shows a confirmation dialog before kicking.
- **Room Code** — displays the room code and QR code again. Includes a copy-to-clipboard button so the host can share it via message/WhatsApp etc.
- **End Game Early** — ends the game immediately and jumps to results with whatever data has been collected so far.

Host cannot go back a round.
