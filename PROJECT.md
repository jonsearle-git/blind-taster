# Blind Taster — Product Specification

## Overview

A food and drink blind testing app for iOS and Android. Clean, polished, tasteful design — no tacky icons. Sold to consumers.

Players join a hosted quiz game and answer a questionnaire once per round. The host sets correct answers, players are scored. Works over the internet via Cloudflare PartyKit — no sign-in required.

- When the host starts a game, the full questionnaire is uploaded to the PartyKit server, which stores it and broadcasts questions to players. The server is the source of truth for all game state.
- All questions are shown on one scrollable screen per round.

---

## Questionnaire

- Questionnaires are created by the host and **stored locally on their device** (SQLite/AsyncStorage)
- Can be created ahead of time and reused across multiple games
- Applied once per round for every round in the game

### Question Types

| Type | Description |
|------|-------------|
| **Multiple choice — text** | Player picks one option from a list of text answers. 1 point if correct. |
| **Multiple choice — number** | Player picks one option from a list of numbers. 1 point if exact match. |
| **Slider / number input** | Same type, UI choice only. Player picks or enters a number. 1 point if exact match. |
| **Tags** | Player selects from a set of text tags (e.g. tasting notes: "grassy", "peach"). Host can limit how many tags a player can select. 1 point per correct tag selected. |
| **Price** | Player enters a monetary value (e.g. £9.99). Currency symbol set by host. 1 point if exact match. |

---

## Rounds

- Host selects the number of rounds (e.g. 5)
- Each round = one item being tested (e.g. Wine A, Wine B)
- Rounds are numbered (1, 2, 3…)
- Host can optionally **label each round** (e.g. "Château Margaux 2015") — this label is **hidden from players** during the game and revealed to players at the end. Host is clearly warned when creating rounds that labels are hidden until reveal.
- **Reminder in the questionnaire builder:** when the host finishes building a questionnaire, show a prompt: "Remember to label the items you are testing in the Rounds screen. These labels are hidden from players until the end."
- Host manually advances players to the next round once everyone has answered
- Host can see per-player response status during a round
- Host can kick any player at any time (player is not told why, just removed)
- Kicked players can rejoin — their name is reserved

---

## Joining

- Players join via a **deep link** (`blindtaster://join/ROOMCODE`) or by entering the room code manually
- Room code is generated by PartyKit
- Host admits or denies each join request
- Denied players see a "You were not admitted into the game" message
- Host can start the game whenever they are ready

---

## Disconnection & Reconnection

- If a player disconnects, the game pauses and waits
- Player can rejoin using the same room code and their reserved name
- If the host disconnects, the game pauses until the host reconnects
- Host can kick unresponsive players to unblock the game

---

## Results

- Once all players have submitted, the host sees two buttons: **Reveal Answers** and **Next Round**
- Host must reveal answers before advancing — the Reveal Answers button appears first, Next Round button appears after reveal
- When host taps Reveal Answers, correct answers are broadcast to all players simultaneously and scores updated
- Player's running score is shown in the banner throughout the game
- At the end: each player sees their final score, finishing position (1st, 2nd, 3rd etc.), right/wrong breakdown per question, and round labels revealed
- Host sees everyone's scores and the winner
- Correct answers are held privately on the server and never sent to players until the host triggers reveal

---


## Host In-Game Navigation

The host navigates between three dedicated screens during the game. All three share a banner with a dropdown in the top-right.

- **HostRoundScreen** — shows per-player answered/waiting status for the current round. Has the advance to next round button (enabled when all players have answered).
- **HostPlayersScreen** — shows all players with a green dot if connected, grey if disconnected. Each player has a kick icon — tapping shows a confirmation dialog before kicking.
- **HostLeaderboardScreen** — shows live scores, updated after each round.

## Host Dropdown Controls

Accessible via a dropdown in the top-right of the banner on all host in-game screens.

- **Room Code** — displays the room code and QR code again. Includes a copy-to-clipboard button so the host can share it via message/WhatsApp etc.
- **End Game Early** — ends the game immediately and jumps to results with whatever data has been collected so far.

Host cannot go back a round.
