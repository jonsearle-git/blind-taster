# Blind Taster — Product Specification

## Overview

A food and drink blind testing app for iOS and Android. Clean, polished, tasteful design — no tacky icons. Sold to consumers.

Players join a hosted quiz game and answer a questionnaire once per round. The host sets correct answers per round, players are scored. Works over the internet via Cloudflare PartyKit — no sign-in required.

- When the host starts a game, the full questionnaire (questions only, no correct answers) and all per-round correct answers are uploaded to the PartyKit server. The server is the source of truth for all game state.
- All questions are shown on one scrollable screen per round.

---

## Questionnaire

- Questionnaires are created by the host and **stored locally on their device** (SQLite)
- Can be created ahead of time and reused across multiple games
- Questions are pure templates — they define the structure (options, tags, range) but carry no correct answers
- Correct answers are set per-round in the Rounds Builder, not in the questionnaire

### Question Types

| Type | Description |
|------|-------------|
| **Multiple choice — text** | Player picks one option from a list of text answers. 100 pts if correct. |
| **Multiple choice — number** | Player picks one option from a list of numbers. 100 pts if exact match. |
| **Slider / number input** | Same type, UI choice only. Player picks or enters a number. Linear partial credit based on proximity. |
| **Tags** | Player selects from a set of text tags (e.g. tasting notes: "grassy", "peach"). Host can limit how many tags a player can select. Proportional credit based on fraction of correct tags chosen. |
| **Price** | Player enters a monetary value (e.g. £9.99). Currency symbol set by host. Linear partial credit based on percentage error. |

---

## Rounds

- Host selects the number of rounds (e.g. 5) in the Rounds Builder
- Each round = one item being tested (e.g. Wine A, Wine B)
- Rounds are numbered (1, 2, 3…)
- Host can optionally **label each round** (e.g. "Château Margaux 2015") — this label is **hidden from players** during the game and revealed to players at the end. Host is clearly warned when entering round details that labels are hidden until reveal.
- **Per-round correct answers:** for each round, the host enters the correct answer to every question. Round 1 might be "Merlot, £20, Cherry+Oak"; Round 2 "Shiraz, £35, Blackcurrant". These are stored server-side and never sent to players until reveal.
- Host manually advances players to the next round once everyone has answered
- Host can see per-player response status during a round
- Host can kick any player at any time (player is not told why, just removed)
- Kicked players can rejoin — their name is reserved

---

## Joining

- Players join via a **deep link** (`blindtaster://join/ROOMCODE`) or by entering the room code manually
- Room code is 8 characters, generated by PartyKit
- Host admits or denies each join request
- Denied players see a "You were not admitted into the game" message
- If a player tries to join a new game while already in a game, they are shown a confirmation dialog
- Host can start the game whenever they are ready

---

## Disconnection & Reconnection

- If the host disconnects, the game pauses until the host reconnects
- If the host is disconnected for 5 minutes (configurable alarm), the game ends automatically
- Host reconnecting after an alarm-triggered game end receives the `game_ended` message on connect
- Players can leave the game at any time via the "Leave Game" button

---

## Results

- Once all players have submitted, the host sees: **Reveal Answers** (appears first) then **Next Round** (appears after reveal)
- Host must reveal answers before advancing
- When host taps Reveal Answers, correct answers are scored server-side and results broadcast simultaneously
- Player's running score is shown in the banner throughout the game
- At the end: each player sees their final score, finishing position (1st, 2nd, 3rd etc.), right/wrong breakdown per question, and round labels revealed
- Host sees everyone's scores and the winner with an expandable per-player breakdown
- Correct answers are held privately on the server and never sent to players until the host triggers reveal

---

## Host In-Game Navigation

The host navigates between three dedicated screens during the game. All three share a banner with a dropdown in the top-right.

- **HostRoundScreen** — shows per-player answered/waiting status for the current round. Has the Reveal Answers and Next Round buttons.
- **HostPlayersScreen** — shows all players with a green dot if connected, grey if disconnected. Each player has a kick icon — tapping shows a confirmation dialog before kicking.
- **HostLeaderboardScreen** — shows live scores, updated after each round.

## Host Dropdown Controls

Accessible via a dropdown in the top-right of the banner on all host in-game screens.

- **Room Code** — displays the room code and QR code again. Includes a copy-to-clipboard button.
- **End Game Early** — ends the game immediately and jumps to results with whatever data has been collected so far.

Host cannot go back a round.

## Screens

### Shared
- HomeScreen — saved questionnaires list with edit/delete, Host and Join buttons

### Host
- SetupGameScreen — pick or create questionnaire
- QuestionnaireBuilderScreen — add/edit/reorder/delete questions (pure templates, no correct answers)
- RoundsBuilderScreen — set round count, optional labels per round, correct answers per round
- HostLobbyScreen — QR code, admit/deny players, start game
- HostRoundScreen — who has answered this round, Reveal Answers / Next Round buttons
- HostPlayersScreen — all players with connection status, kick controls
- HostLeaderboardScreen — live scores after each round
- HostResultsScreen — final breakdown, expandable per-player, Done button

### Player
- JoinGameScreen — room code + name entry, in-game protection
- PlayerLobbyScreen — waiting room, player list, Leave Game button
- PlayerRoundScreen — questions, submit, inline reveal results
- PlayerResultsScreen — position, score, round breakdown, Done button
